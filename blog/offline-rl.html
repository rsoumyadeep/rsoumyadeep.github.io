Offline Reinforcement Learning

IntuitionTheoryAlgorithmsResearch

1. Intuition: Learning Without Interaction

In classical RL, the agent learns by interacting with the environment. In offline RL, the agent is given a fixed dataset and must learn a policy without any further interaction.

Offline RL = Policy learning from a static dataset collected by an unknown behavior policy.

This setting appears naturally in healthcare, robotics, and recommendation systems where exploration is expensive or unsafe.

2. Formal Setup

\( (\mathcal{S}, \mathcal{A}, P, R, \gamma) \)

We observe a dataset:

\( \mathcal{D} = \{(s_i, a_i, r_i, s'_i)\}_{i=1}^N \)

generated by a behavior policy \(\pi_\beta\).

3. Objective

We aim to learn a policy \(\pi\) maximizing:

\( J(\pi) = \mathbb{E}_{\pi} \left[ \sum_{t=0}^{\infty} \gamma^t r_t \right] \)

without access to environment interaction.

4. Core Challenge: Distribution Shift

\( d^{\pi}(s,a) \neq d^{\pi_\beta}(s,a) \)

The learned policy may query actions that are not present in the dataset, causing severe estimation errors.

5. Extrapolation Error (Bootstrapping Issue)

Q-learning relies on bootstrapping:

\( Q(s,a) \leftarrow r + \gamma \max_{a'} Q(s',a') \)

If \(a'\) is out-of-distribution, errors accumulate.

Extrapolation error = evaluating unseen actions using unreliable function approximation.

6. Error Decomposition

Offline RL error can be decomposed into:

  • Approximation error: function class limitation
  • Estimation error: finite data
  • Extrapolation error: distribution mismatch

7. Concentrability Coefficient

A key quantity in theoretical analysis:

\( C = \sup_{s,a} \frac{d^{\pi}(s,a)}{d^{\pi_\beta}(s,a)} \)

If \(C\) is large, learning becomes unstable or impossible.

Good offline RL requires bounded distribution mismatch.

8. Algorithmic Approaches

Behavior Cloning (BC)

Supervised learning on dataset actions.

BCQ

Restricts policy to dataset support.

CQL

Penalizes unseen actions.

IQL

Avoids explicit policy constraints via implicit updates.

9. Conservative Q-Learning (CQL)

\[ \min_Q \; \mathbb{E}_{\mathcal{D}}[(Q - \mathcal{B}Q)^2] + \alpha \left( \mathbb{E}_{a \sim \pi}[Q] - \mathbb{E}_{\mathcal{D}}[Q] \right) \]

This ensures pessimistic value estimates for unseen actions.

11. Minimal PyTorch-style Pseudocode (CQL)

for (s, a, r, s_next) in dataset:
    q = Q(s, a)
    target = r + gamma * max_a Q_target(s_next, a)

    bellman_loss = (q - target)**2

    random_actions = sample_actions()
    conservative_penalty = Q(s, random_actions).mean() - Q(s, a).mean()

    loss = bellman_loss + alpha * conservative_penalty

    optimize(loss)

12. Key Insight

Offline RL balances improvement vs staying close to dataset support.
