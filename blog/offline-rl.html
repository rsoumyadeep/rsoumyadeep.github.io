Offline Reinforcement Learning (Offline RL)

1. Intuition

You are learning purely from a fixed dataset collected by some unknown behavior policy. You cannot explore.

Offline RL = Learning a policy from a static dataset without interaction.

2. Formal Setup

We assume a Markov Decision Process (MDP):

\( (\mathcal{S}, \mathcal{A}, P, R, \gamma) \)

Dataset:

\( \mathcal{D} = \{(s_i, a_i, r_i, s'_i)\}_{i=1}^N \)

Generated by behavior policy \(\pi_\beta\).

3. Bellman Equation (Standard RL)

\[ Q^{\pi}(s,a) = R(s,a) + \gamma \mathbb{E}_{s' \sim P}[ V^{\pi}(s') ] \]

4. Problem in Offline RL

We only have samples from \(\pi_\beta\), but we want to evaluate or optimize a different policy \(\pi\).

Distribution mismatch: \( d^{\pi}(s,a) \neq d^{\pi_\beta}(s,a) \)

5. Extrapolation Error

When Q-learning evaluates unseen actions, it relies on function approximation → leads to overestimation.

Error grows for \( (s,a) \notin \text{support}(\mathcal{D}) \)

6. Conservative Q-Learning (CQL)

Key idea: Penalize Q-values of unseen actions.

\[ \min_Q \; \underbrace{\mathbb{E}_{\mathcal{D}}[(Q(s,a) - \mathcal{B}Q)^2]}_{Bellman\ error} + \alpha \left( \mathbb{E}_{s \sim \mathcal{D}, a \sim \pi}[Q(s,a)] - \mathbb{E}_{\mathcal{D}}[Q(s,a)] \right) \]

7. Interactive Visualization: Distribution Shift

Blue dots = dataset actions, Red dots = policy actions. You will see how the learned policy can move outside dataset support.

8. Key Insight

Good offline RL = Stay close to dataset while improving reward.

9. Research Directions

  • Support-constrained policy optimization
  • Uncertainty estimation
  • POMDPs in offline RL
  • Sample complexity analysis
