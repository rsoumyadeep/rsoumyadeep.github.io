01

Intuition: Why recursion beats brute force

The core idea that turned optimal decision-making into an efficient algorithm

The problem RL solves

An agent lives in an environment. At every step it chooses an action, receives a reward, and moves to a new state. Its goal? Maximize the total discounted future reward over its entire lifetime.

Good future = high value
Bad future = low value

Calculating the total reward for every possible sequence of actions is impossible — the number of paths explodes exponentially.

Bellman’s brilliant insight

Instead of looking at the entire future, just ask:

“What is the best I can get right now plus the best I can expect from tomorrow?”

This recursive decomposition is exactly what the Bellman Equation formalizes. It turns an infinite-horizon problem into a simple one-step lookahead.

Recursive optimality = dynamic programming

Intuition in one sentence:
The optimal value of being in a state today is the immediate reward plus the discounted optimal value of wherever you go tomorrow.

02

Derivation: From definition to equation

Step-by-step from return to Bellman optimality

1. The return (total future reward)

\[ G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1} \]

γ ∈ (0,1) is the discount factor. It makes far-future rewards less important.

2. State-value function under a policy π

\[ V^\pi(s) \triangleq \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] \]

This is the expected return if we start in state s and always follow policy π.

3. One-step unrolling (Bellman Expectation Equation)

Split the return into immediate reward + discounted future return:

\[ V^\pi(s) = \mathbb{E}_\pi \left[ r_{t+1} + \gamma G_{t+1} \mid s_t = s \right] \]

Taking the expectation over the next state and reward (using the transition probabilities of the MDP), we get the Bellman Expectation Equation:

\[ V^\pi(s) = \sum_{a} \pi(a|s) \sum_{s',r} p(s',r \mid s,a) \left[ r + \gamma V^\pi(s') \right] \]
This is a fixed-point equation. The value of today depends only on the values of tomorrow.

4. Optimal policy → Bellman Optimality Equation

The best possible value function V* satisfies:

\[ V^*(s) = \max_a \sum_{s',r} p(s',r \mid s,a) \left[ r + \gamma V^*(s') \right] \]

or in compact expectation form

\[ V^*(s) = \max_a \mathbb{E} \left[ r + \gamma V^*(s') \mid s, a \right] \]
Q-function version (most used in practice):
\[ Q^*(s,a) = \mathbb{E} \left[ r + \gamma \max_{a'} Q^*(s',a') \mid s,a \right] \]
The optimal policy is simply \(\pi^*(s) = \arg\max_a Q^*(s,a)\)
03

Where & how it is used in RL

From classic algorithms to deep RL — the Bellman equation is everywhere

Dynamic Programming

Value Iteration

Repeatedly apply the Bellman optimality backup until convergence:

Vnew(s) ← maxa E [ r + γ V(s′) | s,a ]

Converges to V* → extract optimal policy

Model-free methods

Q-Learning

The famous TD update is a sampled version of the Bellman optimality equation:

\[ Q(s,a) \leftarrow Q(s,a) + \alpha \left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right] \]

No model needed — learns directly from experience

Deep RL & Policy Gradients

Actor-Critic & DQN

Neural networks approximate V* or Q*. The loss function is usually the Bellman error:

\[ \mathcal{L}(\theta) = \mathbb{E} \left[ \left( r + \gamma \max Q_{\theta^-}(s',a') - Q_\theta(s,a) \right)^2 \right] \]

Target network stabilizes training

Every major RL algorithm (SARSA, PPO, SAC, DDPG, TD3, …) is either directly solving or approximating the Bellman equation.

Connection to the Hamilton-Jacobi-Bellman (HJB) Equation

The continuous-state/action/time counterpart

Correction

It’s the Hamilton-Jacobi-Bellman (HJB) equation

You were almost right — it is indeed the continuous version of the Bellman equation, but the standard name is Hamilton–Jacobi–Bellman (HJB) equation.

It arises in optimal control theory for continuous-time dynamical systems with continuous states and actions.

How the discrete Bellman becomes the HJB (limit as Δt → 0)

Discrete Bellman (small time step Δt):
\[ V(s) = \max_u \Big[ r(s,u)\Delta t + (1 - \rho\Delta t) \mathbb{E}[V(s + f(s,u)\Delta t)] \Big] \]
↓ Taylor expansion + limit
HJB equation (infinite-horizon discounted case): \[ \rho V(x) = \min_u \left[ l(x,u) + \nabla_x V(x)^\top f(x,u) \right] \] where:
• \( \rho \): discount rate
• \( l(x,u) \): instantaneous cost/reward
• \( f(x,u) \): continuous dynamics
• \( \nabla_x V(x) \): gradient of the value function
When to use HJB?
Continuous-time control problems (robotics, autonomous driving, finance). Solved analytically only in low dimensions; otherwise approximated numerically.
How RL connects to HJB
Modern continuous-control RL algorithms (DDPG, SAC, TD3) are essentially learning neural-network approximations to the solution of the HJB equation — without ever writing the PDE explicitly.
💡
The Bellman equation is the discrete-time version. The HJB equation is its continuous-time counterpart. Deep RL bridges both worlds by using function approximators to solve the Bellman equation in high-dimensional continuous spaces.
