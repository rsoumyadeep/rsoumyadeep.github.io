01 — Foundations

The Policy Gradient Theorem

Policy gradient methods directly optimize the policy by computing gradients of the expected cumulative reward. Unlike value-based methods, they work natively in continuous action spaces and naturally represent stochastic policies.

🎯

The Objective

Maximize the expected return J(θ) = 𝔼τ~πθ[R(τ)] — the expected sum of discounted rewards under the parameterized policy πθ.

🔗

Policy Parameterization

Policies are parameterized by θ (neural net weights). For discrete actions: softmax. For continuous: Gaussian with learned mean and variance.

🌊

Trajectory Distribution

pθ(τ) = p(s0) ∏t πθ(at|st) p(st+1|st,at). Transitions are part of the environment; we only control the policy factor.

📐

Log-Derivative Trick

∇θ pθ(τ) = pθ(τ) ∇θ log pθ(τ). Converts an intractable gradient of an expectation into an expectation of a gradient — the key identity.

Policy Gradient Theorem — Core Identity
∇θ J(θ) = 𝔼τ~πθ [ Σt ∇θ log πθ(at|st) · Qπ(st, at) ]
// Environment dynamics cancel — gradient depends only on the policy's score function and the Q-values
Score Function Gradient Estimator
∇θ J(θ) ≈ (1/N) Σi Σt ∇θ log πθ(ati|sti) · R(τi)
// Monte Carlo estimate over N rollout trajectories — unbiased but high variance

Derivation: Where Does It Come From?

Write J(θ) as an integral

J(θ) = ∫ pθ(τ) R(τ) dτ. Objective is a continuous sum over all possible trajectories weighted by their probability under the policy.

Differentiate under the integral sign

∇θ J(θ) = ∫ ∇θ pθ(τ) R(τ) dτ. Allowed because reward R(τ) doesn't depend on θ.

Apply log-derivative trick

∇θ pθ(τ) = pθ(τ) ∇θ log pθ(τ). Reintroduce pθ(τ) to convert to an expectation: 𝔼τ~πθ[∇θ log pθ(τ) · R(τ)].

Expand log pθ(τ) — dynamics vanish

log pθ(τ) = log p(s0) + Σt log πθ(at|st) + Σt log p(st+1|st,at). Gradient of constant and dynamics terms is zero — only the policy log-prob survives.

Causality: use future rewards only

Action at can only influence future rewards. Replace R(τ) with the reward-to-go: R̂t = Σt'≥t γt'-t rt'. This does not introduce bias but reduces variance.

02 — Algorithms

Core Algorithms

From vanilla REINFORCE to proximal policy optimization — each algorithm addresses specific shortcomings of its predecessor.

REINFORCE (Williams, 1992)

The original Monte Carlo policy gradient algorithm. Simple and unbiased but suffers from high variance due to full-trajectory returns.

Initialize policy πθ

Randomly initialize neural network parameters θ.

Sample trajectory τ = (s₀,a₀,r₀, …, sT)

Roll out the policy in the environment until termination.

Compute returns Gt

Gt = Σt'=tT γt'-t rt'. Discounted cumulative reward from time t.

Compute policy gradient

∇θ J ≈ Σt ∇θ log πθ(at|st) · Gt

Update parameters

θ ← θ + α ∇θ J. Repeat from step 2.

Loss (negative for gradient ascent via Adam)
ℒ(θ) = -Σt log πθ(at|st) · Gt
// Unbiased estimator. High variance O(T) grows with episode length.
Python · PyTorch
def reinforce_update(policy, optimizer, trajectories, gamma=0.99): loss = 0 for states, actions, rewards in trajectories: returns = compute_returns(rewards, gamma) # G_t for each t log_probs = policy.log_prob(states, actions) loss -= (log_probs * returns).sum() optimizer.zero_grad(); loss.backward(); optimizer.step()

REINFORCE with Baseline

Adding a baseline b(st) that doesn't depend on actions reduces variance without introducing bias. The value function Vπ(st) is the optimal baseline.

Gradient with Baseline
∇θ J(θ) = 𝔼τ [ Σt ∇θ log πθ(at|st) · (Gt - b(st)) ]
// 𝔼[∇θ log π · b] = 0 → baseline does not change the expectation
Why baseline doesn't bias the gradient
𝔼a~π[∇θ log πθ(a|s) · b(s)] = b(s) 𝔼a~π[∇θ log πθ(a|s)]
= b(s) · ∇θ 𝔼a~π[1] = b(s) · ∇θ 1 = 0

Optimal Baseline

The variance-minimizing baseline is b*(s) = 𝔼[‖∇ log π‖² G] / 𝔼[‖∇ log π‖²]. In practice Vπ(s) works very well and is simpler to implement.

Advantage Function

Aπ(s,a) = Qπ(s,a) − Vπ(s). Positive advantage → action is better than average at that state. This is the object Actor-Critic methods learn to estimate.

Trust Region Policy Optimization (Schulman et al., 2015)

Addresses the step-size problem in policy gradients: too large an update can catastrophically degrade the policy. TRPO constrains each update to a trust region via a KL divergence constraint.

TRPO Constrained Optimization
maximizeθ 𝔼t[ πθ(at|st) / πθold(at|st) · Ât ]
subject to 𝔼t[ KL[πθold(·|st) ‖ πθ(·|st)] ] ≤ δ
// δ typically 0.01. Solved via conjugate gradient + line search (expensive)

Collect trajectories with πθold

Gather samples to estimate the surrogate objective and KL constraint.

Compute surrogate objective gradient g

g = ∇θ Lθold(θ) evaluated at θ = θold.

Compute Fisher-vector products

Use conjugate gradient to solve Fx = g where F is the Fisher information matrix (KL Hessian). O(n) per iteration instead of O(n²).

Backtracking line search

Find step size α such that the constraint is satisfied and objective improves.

⚠️ TRPO Limitations

Second-order method: computationally expensive (conjugate gradient solve). Difficult to implement correctly. Does not support shared parameters between actor and critic easily. PPO addresses all of these.

Proximal Policy Optimization (Schulman et al., 2017)

PPO approximates the TRPO constraint using a simpler clipped surrogate objective. First-order method, easy to implement, robust in practice. The de-facto standard for policy optimization.

PPO-Clip Objective
rt(θ) = πθ(at|st) / πθold(at|st) (probability ratio)
ℒCLIP(θ) = 𝔼t[ min( rt(θ) Ât, clip(rt(θ), 1-ε, 1+ε) Ât ) ]
// ε = 0.1 or 0.2. Clipping prevents large policy updates without expensive constrained optimization
Full PPO Loss (with value function and entropy bonus)
ℒPPO(θ) = ℒCLIP(θ) − c₁ ℒVF(θ) + c₂ S[πθ](st)
// c₁ ≈ 0.5 (value coeff), c₂ ≈ 0.01 (entropy coeff to encourage exploration)

When Ât > 0

Action was better than expected. Increase its probability — but clip at (1+ε) to avoid going too far.

When Ât < 0

Action was worse than expected. Decrease its probability — but clip at (1−ε) to avoid going too far in the other direction.

PPO Update Loop
# Collect N rollout steps, then do K epochs of minibatch updates for epoch in range(K): for batch in minibatches(buffer): ratio = exp(new_log_probs - old_log_probs) clip_adv = clamp(ratio, 1-eps, 1+eps) * advantages policy_loss = -min(ratio * advantages, clip_adv).mean() value_loss = 0.5 * (returns - values).pow(2).mean() entropy_bonus = dist.entropy().mean() loss = policy_loss + 0.5*value_loss - 0.01*entropy_bonus optimize(loss)

03 — Variance Reduction

Taming the Variance

High variance is the central challenge of policy gradient methods. Multiple complementary techniques reduce variance while preserving unbiasedness (baselines, causality) or accepting a bias-variance trade-off (GAE).

01Reward-to-Go (Causality)
Replace full trajectory return R(τ) with reward-to-go R̂t = Σt'≥t γt'-t rt'. The causality principle: action at cannot affect rewards before time t, so including those rewards only adds noise. This is a zero-bias reduction — the gradient estimate is still unbiased but variance decreases because we remove irrelevant randomness from the past.
02Baselines (State-Dependent)
Any function b(st) can be subtracted from the return without changing the expected gradient: 𝔼[∇θ log π · b(s)] = 0. The optimal baseline (minimum variance) is b*(s) = 𝔼[‖∇ log π‖² G] / 𝔼[‖∇ log π‖²]. In practice, we use Vπ(st) — the learned state value function — which serves double duty as a zero-bias variance reducer and a critic.
03Generalized Advantage Estimation (GAE)
GAE (Schulman et al., 2016) interpolates between high-variance Monte Carlo returns and low-variance but biased 1-step TD via an exponentially weighted average of k-step advantages:

ÂGAE(γ,λ)t = Σl=0∞ (γλ)l δt+l

where δt = rt + γ V(st+1) − V(st) is the TD residual. λ=0: pure 1-step TD (low variance, high bias). λ=1: Monte Carlo (high variance, no bias). λ ∈ (0,1) trades off. Typically λ = 0.95.
04Reward Normalization & Whitening
Normalize advantages within a mini-batch: Â ← (Â − mean(Â)) / (std(Â) + ε). This is a heuristic (not theoretically motivated like the others) but is crucial in practice: prevents reward scale from affecting step size, stabilizes training. Used in PPO, RLHF pipelines universally.
05Control Variates & Importance Sampling
Importance sampling allows off-policy data reuse: wt = πθ/πold. Allows multiple gradient steps per batch of data (as in PPO). The clipping in PPO can be seen as a truncated importance weight. In RLHF, off-policy rollouts from the SFT model are sometimes reused with IS correction.

A2C — Advantage Actor-Critic

Synchronous version of A3C. Multiple parallel environments collect experience; a central gradient update step uses advantages estimated by the critic. Stable training, easy parallelism.

A3C — Async Advantage Actor-Critic

Asynchronous gradient updates from N workers. Each worker maintains a copy of the policy, computes gradients, pushes to shared network. Lock-free. Faster wall-clock but noisier gradients.

SAC — Soft Actor-Critic

Off-policy actor-critic with maximum entropy objective: J = 𝔼[Σ γt(rt + α H(π))]. Automatically tunes temperature α. State-of-the-art for continuous control.

TD3 — Twin Delayed DDPG

Addresses Q-function overestimation in DDPG with twin critics (take min), delayed policy updates, and target policy smoothing. Deterministic policy, off-policy, very sample-efficient.

Actor-Critic Update Rules
δt = rt + γ Vw(st+1) − Vw(st) (TD error ≈ advantage)
θ ← θ + αactor ∇θ log πθ(at|st) · δt
w ← w − αcritic δt · ∇w Vw(st) (MSE gradient)
05 — Advanced Topics

Advanced Concepts

Natural Policy Gradient

Standard gradient ascent is invariant to Euclidean geometry in parameter space, not the distribution space. The natural gradient uses the Fisher Information Matrix (FIM) to follow the steepest ascent in distribution space.

Natural Gradient Update
θ ← θ + α F(θ)⁻¹ ∇θ J(θ)
F(θ) = 𝔼s,a~π[ ∇θ log πθ(a|s) ∇θ log πθ(a|s)ᵀ ]
// F is the Fisher Info Matrix = KL Hessian. TRPO approximates F⁻¹ via conjugate gradient.

The natural gradient is invariant to reparameterization of the policy — it measures progress in terms of change in the policy distribution, not parameter change. TRPO and K-FAC approximate the natural gradient efficiently.

Maximum Entropy RL & Entropy Regularization

Add an entropy bonus to the objective to encourage exploration and prevent premature convergence to deterministic policies.

Max-Entropy Objective
JMaxEnt(θ) = 𝔼τ[ Σt γt (rt + α H(πθ(·|st))) ]
H(π(·|s)) = −𝔼a~π[log π(a|s)] (policy entropy at state s)
// α: temperature. Higher α → more stochastic policy. SAC auto-tunes α.

Benefits

Encourages exploration, leads to more robust policies, avoids deterministic optima, better generalization. Foundation of SAC and connects to energy-based models.

In RLHF

Entropy bonus in PPO during RLHF fine-tuning prevents over-optimization and mode collapse, similar to the KL penalty between the fine-tuned and reference policy.

GRPO — Group Relative Policy Optimization (DeepSeek, 2024)

GRPO eliminates the value function critic entirely. Instead, advantages are computed by comparing rewards across a group of sampled outputs for the same question — much cheaper for LLMs.

GRPO Advantage Estimation
Sample G outputs {o1, …, oG} for question q from πθold
Âi = (ri − mean({rj})) / std({rj}) (group-normalized reward)
// No critic network needed. Naturally handles sparse rewards from verifiers.
GRPO Objective
ℒGRPO(θ) = 𝔼q,o[ min(r·Â, clip(r,1-ε,1+ε)·Â) ] − β KL[πθ‖πref]
// β-KL regularization replaces value function loss. Used in DeepSeek-R1, Qwen GRPO.

Why GRPO for LLMs?

Training a value function that operates on tokens is expensive and tricky. For math/code tasks with binary verifiable rewards, GRPO's group-normalized reward naturally provides relative advantage signal without a separate critic model.

Off-Policy Policy Gradients

Reusing data from old policies or a different behavior policy via importance sampling. Enables higher sample efficiency at the cost of variance from IS weights.

Off-Policy Gradient Estimator
∇θ J(θ) ≈ 𝔼τ~πβ[ ∏t'≤t(πθ/πβ) · ∇θ log πθ(at|st) · Qπβ(st,at) ]
// Product of IS weights can have exponential variance over long horizons — clip or per-step IS

PPO uses a truncated 1-step IS ratio (the clipping). V-trace (IMPALA) uses per-step truncated IS for distributed training. Retrace uses off-policy safe return targets.

1

Supervised Fine-Tuning (SFT)

Train the base LLM on high-quality human-written demonstrations using standard cross-entropy loss. Produces πSFT — a capable instruction-following model, but not yet aligned. This is the initialization point for both the RL actor and the reference policy.

2

Reward Model (RM) Training

For a given prompt x, sample multiple completions {y1, …, yk}. Human annotators rank them. Fit a scalar reward model rφ(x,y) by minimizing the Bradley-Terry pairwise ranking loss:

Reward Model Loss
ℒRM(φ) = −𝔼(x,yw,yl)[ log σ(rφ(x,yw) − rφ(x,yl)) ]
// yw: preferred, yl: rejected. σ: sigmoid. Trains RM to assign higher scalar to preferred outputs.

The RM is typically initialized from the SFT model with a linear scalar head replacing the LM head.

3

PPO Fine-Tuning with KL Regularization

The actor (πθ, initialized from SFT) generates completions. The reward model scores them. A frozen reference policy πref (= πSFT) provides KL regularization to prevent reward hacking and catastrophic forgetting:

RLHF Reward with KL Penalty
R(x,y) = rφ(x,y) − β · KL[πθ(·|x) ‖ πref(·|x)]
= rφ(x,y) − β · Σt log(πθ(yt|x,y) / πref(yt|x,y))
// β typically 0.01–0.1. KL term penalizes drift from SFT — prevents reward hacking.

PPO then maximizes 𝔼x~D, y~πθ[R(x,y)] using GAE advantages and clipped surrogate objective. Value function is a separate copy of the LM initialized from SFT or RM.

4

Reward Hacking & Over-Optimization

As PPO optimizes against rφ, the policy can find out-of-distribution completions that fool the reward model without being genuinely good. The KL penalty, entropy regularization, and periodic reward model updates (RLHF loops) mitigate this. The gold reward typically follows a concave relationship with the RM score — there is an optimal stopping point.

Alternative: Direct Preference Optimization (DPO)

DPO — No RL Loop

DPO shows that the optimal RLHF policy has a closed form. The reward model can be implicitly reparameterized in terms of policy ratios, eliminating the RL loop entirely and training directly on preferences with a simple classification loss.

DPO Loss

ℒDPO(θ) = −𝔼[ log σ( β log(πθ(yw|x)/πref(yw|x)) − β log(πθ(yl|x)/πref(yl|x)) ) ]

DPO vs PPO

DPO: simpler, stable, no RM training. PPO: more flexible, supports online data collection, better with process reward models and verifiable tasks. Current trend leans PPO for reasoning.

07 — LLM Post-Training

RL in LLM Post-Training

Modern frontier LLMs like GPT-4, Claude, Gemini, and DeepSeek-R1 all use RL extensively during post-training. Here's the complete picture — what gets trained, what reward signal is used, and how policy gradient methods fit in.

🎓

Phase 1: Pre-training

Next-token prediction on massive corpora. No RL. Produces a powerful but unaligned base model. The starting point for all post-training.

📝

Phase 2: SFT

Supervised fine-tuning on instruction–response pairs. Teaches the model the format and style of helpful responses. Initializes πSFT = πref.

🏆

Phase 3: RM Training

Train reward model on human preference comparisons (or AI feedback — RLAIF). Produces a scalar oracle for the RL phase.

🔄

Phase 4: RL (PPO/GRPO)

Policy gradient optimization of πθ against reward model signal + KL penalty. Produces the final aligned, helpful, harmless model.

The RL Loop in LLM Training — Mechanics

→Rollout Generation
The actor πθ autoregressively samples completions y = (y1,…,yT) for prompts x sampled from the dataset. In token-level RL, each token generation is an action, the partial sequence is the state, and the reward is typically sparse at the end of sequence (EOS). Generating long sequences with PPO is expensive — this is why efficient rollout engines (vLLM, TensorRT-LLM) matter.
→Token-Level MDP Formulation
State st = (x, y1:t-1): the context + partial generation. Action at = yt ∈ vocabulary (|V| ~ 32K-128K). Transition: deterministic (next token appended). Reward: rT = rφ(x,y) at EOS, else 0 (plus per-token KL penalty: −β log πθ/πref). Horizon T can be 256–32K tokens — very long, requiring careful discount factor (γ ≈ 1).
→Value Function Challenges in LLMs
The critic Vw(st) must be a full LLM that reads the partial sequence and predicts expected return. This doubles memory requirements. The value network is typically warm-started from the RM or SFT model. GRPO bypasses this entirely by using group-normalized rewards. Process Reward Models (PRMs) provide denser supervision at intermediate steps (e.g., math chain-of-thought).
→Process Reward Models (PRMs)
For reasoning tasks (math, code), outcome-only rewards are very sparse. PRMs assign reward to each reasoning step. This provides denser gradient signal and enables MCTS-based search at test time. DeepSeek uses process reward models for R1. OpenAI's PRM800K dataset trains step-level verifiers for math. PRMs can be combined with Monte Carlo Tree Search for best-of-N selection.
→RLAIF — RL from AI Feedback
Instead of human annotators, a more powerful AI (teacher LLM) provides the preference labels. Constitutional AI (Anthropic) generates critiques and revisions, then trains an RM on AI comparisons. Scalable supervision: reduces human annotation cost while preserving the RLHF framework. The policy gradient mechanism is unchanged — only the reward signal source shifts.
→Iterative / Online RLHF
The reward model can be periodically retrained on data from the current πθ to prevent distribution shift and reward hacking. Online DPO and Self-Play Fine-Tuning (SPIN) generate rejected responses from the current policy. Rejection Sampling Fine-Tuning (RFT) generates multiple completions, keeps those passing a verifier, and trains on them — a form of best-of-N filtered SFT, bridging RLHF and supervised training.

Systems View: Engineering PPO for LLMs

4 Model Problem

PPO-RLHF requires simultaneous forward passes for: (1) actor πθ, (2) reference πref, (3) critic Vw, (4) reward model rφ. 4× memory of SFT. Solved with LoRA, offloading, or sharing weights.

KL Budget

KL divergence from SFT must be monitored. Too much → catastrophic forgetting and reward hacking. Too little → insufficient alignment. Target KL ~5–10 nats typical. Adaptive β controllers exist.

Sequence-Level vs Token-Level

InstructGPT uses sequence-level reward + per-token KL. Process reward models assign per-step rewards. Token-level advantages require careful credit assignment — actions (tokens) deep in sequence receive very small gradient signal.

Distributed Training

Rollouts are generated with optimized inference (vLLM). Gradient updates use FSDP / Megatron. Separate rollout and training clusters communicate via a data buffer. Ray + DeepSpeed common stack.
