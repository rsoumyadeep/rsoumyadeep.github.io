01 — Foundation

Why Function Approximation?

Tabular RL requires storing one value per state (or state-action pair). For real-world problems — continuous spaces, pixel inputs, robotic states — this is completely intractable. Function approximation replaces the lookup table with a parameterized function.

∞

Curse of Dimensionality

A robot arm with 7 joints has exponential state space. A 64×64 Atari frame has ~109864 states — physically impossible to enumerate or store.

⊕

Generalization

Similar states should have similar values. A parameterized function maps nearby inputs to nearby outputs, enabling generalization — learning from one state improves estimates of similar states.

⟨⟩

Compact Representation

Instead of |S| parameters, approximate with d ≪ |S| parameters θ ∈ ℝd. The inductive bias of the function class encodes regularity assumptions about the value landscape.

∇

Differentiable Optimization

With differentiable approximators (linear, neural), gradient-based optimization applies directly. This connects RL to the rich toolkit of deep learning and convex optimization.

The Core Idea

Parameterized Value Functions

Replace the tabular state-value function V: S → ℝ with a parameterized approximation V̂(s; θ) where θ ∈ ℝd is a weight vector with d ≪ |S|.

// State-value approximation
V̂(s; θ) ≈ Vπ(s)

// Action-value approximation
Q̂(s, a; θ) ≈ Qπ(s, a)

// Policy parameterization
π(a|s; θ) — stochastic policy
Key insight: Updating θ changes V̂ at ALL states simultaneously — unlike tabular methods. This is both the power (generalization) and the danger (instability) of function approximation.

02 — Linear Methods

Linear Function Approximation

The simplest and theoretically most tractable class of approximators. Despite being linear in parameters, the choice of feature map φ(s) determines expressive power.

// Linear in parameters, nonlinear in state via features
V̂(s; θ) = θ⊤φ(s) = Σi=1..d θi · φi(s)

// Gradient is simply the feature vector
∇θ V̂(s; θ) = φ(s)

// Semi-gradient TD(0) update
θ ← θ + α · [R + γV̂(S'; θ) − V̂(S; θ)] · φ(S)
Feature Representations

Polynomial Features

For state s, polynomial features of order k include all monomials s₁c₁...sₙcₙ where Σcᵢ ≤ k.

φ(s) = [1, s, s², ..., sn]⊤

φ(s₁,s₂) = [1, s₁, s₂, s₁s₂, s₁², s₂², ...]⊤
⚠️ Polynomial features suffer from numerical instability at high degrees. Use orthogonal polynomials (Legendre, Chebyshev) for better conditioning.
Polynomial Basis Functions φᵢ(s) = sⁱ

Fourier Cosine Basis

Cosine features of increasing frequency. Particularly effective for periodic value functions. Different features benefit from different learning rates.

φi(s) = cos(πci⊤s)

ci ∈ {0,1,...,n}k — integer vector

// Recommended: per-feature αi = α/‖ci‖
Fourier Basis Functions

Tile Coding

Multiple offset tilings of the state space. State s activates exactly one tile per tiling. Binary sparse features — computationally very efficient.

φ(s) ∈ {0,1}n·|tiles| — sparse binary

V̂(s;θ) = Σj: φj(s)=1 θj

// Only n features active → O(n) update cost
✓ Tile coding is memory-efficient and fast. Classic RL benchmarks (Mountain Car, Acrobot) use it extensively.

Radial Basis Functions

Gaussian bumps centered at prototype states. Smooth generalization. Connects to kernel regression and RKHS theory.

φi(s) = exp(−‖s − ci‖² / 2σi²)

V̂(s;θ) = Σi θi · exp(−‖s − ci‖² / 2σ²)

// Kernel perspective: RKHS, Gaussian process

Semi-Gradient Methods

Important: RL uses semi-gradient (not full gradient) updates because the TD target itself depends on θ. The gradient through the target is ignored, making the update a biased estimator. This is a crucial distinction from supervised learning.
// Mean Squared Value Error (objective)
J(θ) = Σs∈S μ(s) [vπ(s) − V̂(s;θ)]²

// μ(s) = on-policy stationary distribution of π

// MC update: unbiased, high variance
θ ← θ + α [Gt − V̂(St;θ)] ∇V̂(St;θ)

// TD(0) semi-gradient: biased, low variance
θ ← θ + α [Rt+1 + γV̂(St+1;θ) − V̂(St;θ)] ∇V̂(St;θ)

⟂

Universal Approximation

A single hidden layer with enough neurons approximates any continuous function on a compact set (Cybenko 1989). Deeper networks are exponentially more efficient for certain function classes.

∂

Backpropagation & Semi-Grad

Autodiff computes ∇θV̂(s;θ) efficiently via chain rule. In RL, gradient flows through network but NOT through the bootstrap target — the semi-gradient distinction.

⊗

Dueling Architecture

Q(s,a;θ) = V(s;θ) + A(s,a;θ) − mean_a A(s,a;θ). Separates state value from advantage. Faster learning when actions don't matter much.

// Q-network: maps state → Q-values for all actions
h0 = φ(s) — input (pixels, features)
hl = σ(Wlhl-1 + bl) — hidden layers l=1..L-1
Q̂(s,·;θ) = WLhL-1 + bL — one value per action

θ = {W1, b1, ..., WL, bL} — all parameters
04 — Algorithms

Key Algorithms

TD(λ) with Function Approximation

TD(λ) bridges Monte Carlo (λ=1) and TD(0) (λ=0) via eligibility traces. The trace vector z accumulates gradients, efficiently propagating credit backwards through time.

// TD error
δt = Rt+1 + γV̂(St+1;θ) − V̂(St;θ)

// Accumulating eligibility trace
zt = γλzt-1 + ∇V̂(St;θ)

// Dutch traces (preferred, linear FA)
zt = γλzt-1 + (1−αγλzt-1⊤φt)φt

θt+1 = θt + αδtzt
λ interpolates bias-variance: TD(0) has low variance but bootstrap bias; MC is unbiased but high variance. Optimal λ is problem-dependent.
λ Effect: Weight on n-Step Returns
Lambda λ0.50

Deep Q-Network (DQN)

Mnih et al. (2015). Key innovations: experience replay and target network. First deep RL system to master diverse Atari games from raw pixels.

// Two networks: online θ and target θ⁻
yj = rj + γ · maxa' Q̂(sj', a'; θ⁻) — TD target

L(θ) = 𝔼(s,a,r,s')~D[(y − Q̂(s,a;θ))²]

θ ← θ − α∇θL(θ) — Adam/RMSprop

θ⁻ ← θ — periodic copy every C steps

Experience Replay

Store (s,a,r,s') in buffer D. Sample random minibatches. Breaks temporal correlations, enables data reuse, improves sample efficiency.

Target Network

Separate θ⁻ for computing targets. Updated every C steps. Keeps TD target stationary during gradient updates, stabilizing training.

ε-Greedy Exploration

With prob ε select random action, else argmax Q. Anneal ε from 1.0 → 0.1. Balances exploration and exploitation.

Policy Gradient Methods

Directly parameterize and optimize the policy π(a|s;θ). The Policy Gradient Theorem gives the exact gradient without needing to differentiate the state distribution.

// Policy Gradient Theorem (Sutton et al. 1999)
∇θ J(θ) ∝ Σs μπ(s) Σa Qπ(s,a) ∇θ π(a|s;θ)

// Log-derivative trick (REINFORCE)
= 𝔼π[Qπ(St,At) ∇θ log π(At|St;θ)]

// REINFORCE with baseline (variance reduction)
θ ← θ + α (Gt − b(St)) ∇θ log π(At|St;θ)
The baseline b(s) can be any function of state — it doesn't bias the gradient in expectation. Using V̂(s) gives the advantage Aπ(s,a) = Qπ(s,a) − Vπ(s).

Actor-Critic Methods

Two function approximators: actor π(a|s;θ) and critic V̂(s;w). The critic estimates the baseline/advantage, reducing variance while maintaining low bias.

// Actor: policy parameters θ
θ ← θ + αθ δt ∇θ log π(At|St;θ)

δt = Rt+1 + γV̂(St+1;w) − V̂(St;w)
// Critic: value parameters w
w ← w + αw δt ∇w V̂(St;w)

// A3C: async parallel actors
// PPO: clipped surrogate objective
// SAC: maximum entropy critic

A2C / A3C

Advantage Actor-Critic. A3C uses asynchronous parallel actors to decorrelate experience without replay. On-policy, recurrent-policy-friendly.

PPO (Schulman 2017)

Clipped surrogate: LCLIP = 𝔼[min(r̂tÂ, clip(r̂t,1±ε)Â)]. Simple, robust. Dominates RLHF and robotics.

SAC (Haarnoja 2018)

Maximum entropy: J(π) = 𝔼[Σ rt + αH(π(·|st))]. Off-policy, excellent sample efficiency, robust hyperparameters.

Least-Squares TD (LSTD)

Finds the TD fixed point in closed form via a linear system solve. More data-efficient than incremental TD but O(d²) memory and O(d³) computation per update.

// Fixed point: θ = A⁻¹b
A = Σt φ(St)[φ(St) − γφ(St+1)]⊤
b = Σt Rt+1 φ(St)

// LSTD(λ): extend with eligibility traces
// LSPI: LSTD policy eval + greedy improvement
LSPI (Lagoudakis & Parr, 2003) alternates LSTD policy evaluation with greedy policy improvement. Provably convergent for linear FA, far more sample-efficient than incremental TD.
05 — Theory

Convergence & The Deadly Triad

Not all combinations of function approximation, bootstrapping, and off-policy learning converge. The deadly triad is the central challenge of deep RL theory.

The Deadly Triad (Sutton & Barto 2018)

Instability arises when all three combine:

1. Function Approximation

Any parameterized approximator. Tabular methods are the only fully convergent setting — they are exact representations.

2. Bootstrapping

Using estimated values as targets (TD, DP). MC avoids this by using full returns. Bootstrap bias compounds with approximation error.

3. Off-Policy Training

Distribution mismatch between behavior policy μ and target policy π. The update distribution ≠ stationary distribution used in MSVE.

06 — Modern Deep RL

State-of-the-Art Methods

Mnih et al., 2015
DQN — Human-Level Atari
Deep Q-Network with experience replay and target network. First demonstration of deep RL mastering diverse domains from raw pixels. Nature paper.
Van Hasselt et al., 2016
Double DQN — Overestimation Fix
Decouple action selection (online net θ) from action evaluation (target net θ⁻): y = r + γQ(s', argmax_a Q(s',a;θ); θ⁻). Reduces systematic overestimation.
Schaul et al., 2016
Prioritized Experience Replay
Sample transitions proportional to |δ_t|^α. IS weights ∝ (1/N·1/p_i)^β correct for sampling bias. More efficient use of experience.
Schulman et al., 2017
PPO — Proximal Policy Optimization
LCLIP(θ) = 𝔼[min(r_t(θ)Â_t, clip(r_t(θ),1±ε)Â_t)]. Simple clipped objective prevents destructive updates. Dominates RLHF and robotics.
Haarnoja et al., 2018
SAC — Soft Actor-Critic
Maximum entropy RL: J(π) = Σ𝔼[r_t + αH(π(·|s_t))]. Off-policy, sample efficient, robust to hyperparameters. State-of-the-art continuous control.
Hessel et al., 2018
Rainbow — Combining DQN Improvements
Integrates: Double Q, Prioritized ER, Dueling, Multi-step returns, Distributional RL, NoisyNets. Synergistic combination massively outperforms individual components.
2022–Present
Transformers, World Models, Foundation RL
Decision Transformer, Gato, DreamerV3 (world models). Large-scale pretraining for RL. The frontier: offline RL, multi-task, hierarchical, and language-conditioned policies.
Advanced Topics

Research Frontiers

𝒟

Distributional RL

Model full return distribution Z(s,a) instead of just 𝔼[Z]. C51, QR-DQN, IQN. Better risk-sensitive behavior, improved empirical performance, richer learning signal.

ψ

Successor Representations

M(s,s') = 𝔼[Σ γt𝟙(S_t=s')|S_0=s]. Factorizes into SR (dynamics) × reward weights. Fast adaptation when reward changes; connects to neuroscience (hippocampal place cells).

⟨z⟩

Representation Learning

Learning φ(s) end-to-end: auxiliary tasks (SR, reward prediction), contrastive learning (CURL, SPR), bisimulation metrics. Critical for pixel-based RL and transfer.

κ

Neural Tangent Kernel

Infinite-width NNs ≡ kernel methods with NTK. Training in lazy regime resembles linear FA. Explains generalization and informs network architecture choices.

ω

Natural Policy Gradient

Precondition gradient with Fisher information F(θ): θ ← θ + αF⁻¹(θ)∇J(θ). Parameterization-invariant. TRPO uses constrained NPG; K-FAC approximates F efficiently.

∮

Offline RL

Learning from fixed datasets without environment interaction. Challenges: distribution shift, overestimation of OOD actions. CQL, IQL, TD3+BC address this with pessimistic value estimates.
